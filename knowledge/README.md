# Knowledge

Reference documentation for payments, system design, and career planning.

## What's Here

### payments-fundamentals/

Core payments concepts: how money moves, what can go wrong, how to prove correctness.

This is the domain I'm building expertise in for Staff Engineer.

### system-fundamentals/

Production infrastructure knowledge: Kubernetes, observability, reliability, networking.

Useful for system design interviews and understanding how Weave's systems work.

### system-design/

Design patterns and tradeoffs. Cheat sheets for common architectural decisions.

### career-path/

The plan for Staff → EM → Senior EM → Director. Includes the 52-week payments specialist plan.

## How I Use This

When I learn something, I write it down here. Writing forces clarity. If I can't explain it simply, I don't understand it yet.

These docs are reference material, not tutorials. They assume I've already done the work to understand the concept.
