# Projects

Working code that implements payments concepts.

## What's Here

### payer-sync

Healthcare payment reconciliation system. Ingests ERA and VCC files, reconciles them, processes payments, writes back to PMS.

**Concepts covered**: Reconciliation, state machines, idempotency, audit trails, failure handling

**Status**: Done (May 2026)

### ledger-db

Double-entry ledger implementation in Postgres. Accounts, transactions, entries, balances.

**Concepts covered**: Double-entry bookkeeping, atomic operations, immutability, idempotency

**Status**: Core complete, can extend with more scenarios

### will-they-pay

ML model predicting payment likelihood. Built for Weave hackathon.

**Concepts covered**: Feature engineering, model evaluation, connecting predictions to business decisions

**Status**: Done (June 2026). Document as case study, not career direction.

## Why Projects Over Exercises

Small exercises didn't stick. I'd complete one, feel a small sense of accomplishment, and forget it by the next week.

Projects forced me to solve real problems. The learning stuck because I had to make decisions, handle edge cases, and think about how pieces fit together.

Each project here taught me more than a month of 15-minute exercises would have.
